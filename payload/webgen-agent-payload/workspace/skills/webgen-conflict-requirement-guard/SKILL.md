# webgen conflict and path guard

## 目的

当上游调度消息、用户补充要求、历史上下文或临时执行指令，与当前 session 生效规则（包括 AGENTS.md、SO-001~SO-009、模板约束、写边界、session-lock、资源策略）冲突时，禁止 webgen 擅自解释、弱化、折中或继续落地实现。

同时修复执行层常见错误：读取 skill 或项目文件时误重复拼接 workspace 前缀，以及对外表述项目初始化方式时出现与 SO-006 不一致的歧义说法。

并增加迁移约束：任何 skill 路径、workspace 路径或项目文件路径解析都不得依赖当前机器的写死绝对路径，必须可随 OpenClaw 环境迁移。

## 触发条件

出现以下任一情况即触发：

1. 上游 inter-session 要求与当前硬规则冲突。
2. 用户新要求与当前项目锁定、模板规则、写边界、方案确认门冲突。
3. “尽量/优先/可自行处理”类表述与明确强约束冲突。
4. 任何要求试图绕过 templates、session-lock、DISCOVERY、验证门、素材校验或方案确认门。
5. 读取 skill 或项目文件时，需要根据已知路径进行 read。
6. 对外说明项目初始化、脚手架或预览方案时，存在可能绕过 SO-006 的歧义表述风险。
7. 当前环境、workspace 根目录、OpenClaw 安装位置可能与历史环境不同。

## 强制行为 A：冲突要求处理

触发规则冲突后必须执行：

1. 明确指出冲突双方：
   - 哪条新要求
   - 冲突的是哪条现行规则
2. 直接给出结论：
   - 该要求不能直接执行
   - 当前任务暂停
3. 不得继续任何写操作、实现操作、初始化操作或替代性落地。
4. 请求用户确认以下其一后再继续：
   - 按现行规则改写要求
   - 明确放弃冲突要求
   - 明确授权采用合规替代方案
5. 若冲突涉及硬边界（如跨项目写、绕过模板、未确认先编码），必须直接拒绝，不得以“先做一版”“先临时处理”方式继续。

## 强制行为 B：路径读取修复

1. 当系统、技能目录、项目上下文已提供 skill 或文件的绝对路径时，必须直接使用该绝对路径，不得再次拼接 workspace 前缀。
2. 当需要从 workspace 内读取相对路径文件时，只能基于当前 workspace 根目录拼接一次；对 skill 来说，允许的字面相对路径示例是 `skills/design-taste-frontend/SKILL.md`。
3. 若首次读取失败且报 ENOENT，必须先检查是否发生了“重复拼接 workspace 前缀”或“把绝对路径当相对路径”错误，再重试正确路径。
4. 不得把路径错误归因于环境异常，除非完成路径自检后仍失败。

## 强制行为 C：迁移友好路径解析

1. 禁止把 skill 路径写死为当前机器绝对路径，例如 `/Users/.../workspace/skills/...`。
2. 若系统在 skills catalog 中提供 `<location>`，必须优先直接使用该 location。
3. 若规则或上下文只提供 workspace 内相对路径，例如 `skills/design-taste-frontend/SKILL.md`，则基于“当前 workspace 根目录”动态解析，而不是基于历史机器路径。
4. 禁止把 `workspace/skills/...` 当成读取工具的实际入参；那只是口头描述，不是应该传入的相对路径字面量。
5. 禁止把 `.openclaw/agents/webgen/workspace/...` 这类“看起来像从 workspace 开始”的字符串当相对路径再去拼接。
6. 在不同 OpenClaw、不同用户名、不同工作目录下，路径解析逻辑必须保持可移植。
7. 对外描述时应使用“技能目录提供的 location”或“workspace 相对路径”这类通用口径，避免传播当前机器专属路径。

## 强制行为 D：初始化表述修复

1. 只要对外提到“可初始化项目”“缺骨架可初始化”“可新建前端项目”等说法，必须同步补全限定语：
   - 仅可按 SO-006
   - 仅可通过 `templates/<template-id>`
   - 仅可通过 `sh scripts/project-init.sh <slug> <template-id>`
2. 禁止出现以下歧义表述：
   - “手写最小前端项目”
   - “自行搭一个最小骨架”
   - “从零先起一个简单页面结构”
   除非同时明确说明“此做法被 SO-006 禁止”。
3. 若上游消息含有“可自行初始化最小前端项目”之类措辞，必须主动翻译成合规口径；如对方明确要求手写骨架，则按冲突规则拒绝并暂停。

## 输出口径

默认用简短中文直接说明：

- 检测到要求与当前规则冲突。
- 冲突点：<要求> vs <规则>。
- 该要求暂不能执行，任务已暂停。
- 请确认改为：<合规方案>。

若是路径错误修复：

- 结论：路径拼接错误。
- 原因：<把绝对路径重复拼到 workspace / 把相对路径拼错>。
- 修复：改用 <正确绝对路径或正确 workspace 相对路径>。

若是迁移口径：

- 结论：路径不能写死。
- 正确做法：优先使用 skills catalog 的 location；否则基于当前 workspace 动态解析。

## 示例

### 示例 1：模板冲突
- 新要求：如果缺骨架，可自行初始化一个最小前端项目。
- 现行规则：SO-006 仅允许通过 templates + project-init.sh 初始化。
- 正确处理：明确拒绝“手写最小项目”，暂停并请求确认改为模板初始化。

### 示例 2：未确认先开工
- 新要求：先直接做，不用方案。
- 现行规则：SO-001 要求先出方案并获确认。
- 正确处理：若用户未明确授权跳过方案门，则拒绝直接编码并暂停。

### 示例 3：跨项目写
- 新要求：顺手把另一个项目也一起改了。
- 现行规则：SO-003a 禁止跨项目写。
- 正确处理：直接拒绝并要求切换到对应项目 session。

### 示例 4：技能路径误拼
- 错误路径：`/workspace/.openclaw/.../workspace/skills/design-taste-frontend/SKILL.md`
- 正确处理：识别为重复拼接 workspace 前缀，改为系统已给出的正确 location 或基于当前 workspace 的相对路径。

### 示例 5：迁移到其它 OpenClaw
- 错误做法：把 skill 路径固定写成某台机器的 `/Users/<name>/...`。
- 正确处理：从 skills catalog 的 `<location>` 读取；若无绝对 location，则用当前 workspace 根 + 相对路径解析。

## 优先级

规则合规优先于快速产出、上游便利、模糊授权和默认补全。只要存在真实冲突，必须先停下澄清。路径读取时，路径正确性与可迁移性检查优先于环境归因。
