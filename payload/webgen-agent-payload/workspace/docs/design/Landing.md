## 宣传类页面方案模板

1. 页面类型 / 目标
   - 单页转化型 Landing，目标是让用户在首屏 3 秒内理解价值，并完成注册 / 留资 / 咨询。
2. 目标用户 / 使用场景
   - 首次访问用户，以移动端和桌面浏览为主，决策时间短，需要快速建立信任。
3. 主要板块与信息层级
   - Hero
   - 核心卖点
   - 场景 / 功能说明
   - 社会证明 / 案例
   - FAQ
   - CTA 收口
4. 设计拨盘
   - `DESIGN_VARIANCE = 6-8`
   - `MOTION_INTENSITY = 4-6`
   - `VISUAL_DENSITY = 3-4`
5. 配色 / 字体 / 图标
   - 中性底色 + 1 个主强调色
   - 标题使用更有识别度的无衬线
   - 图标默认 Lucide
6. 素材 / 文案 / 接口
   - 用户提供文案优先；无文案时先做结构占位并标明待补
   - 若有表单提交，默认考虑本地代理或 mock
7. 技术实现
   - 优先原生或延续现有项目技术栈
   - 动效先用 CSS / Anime.js；滚动叙事再升级 GSAP
8. 状态设计
   - 表单提交 `loading / success / error`
   - 无案例或无 FAQ 时给安全降级文案
9. 响应式策略
   - H5 首屏先露出标题、价值点、主 CTA
   - Pad 保持两段式信息组织，避免桌面布局硬缩
   - 触屏 hover 信息改默认可见或点击展开
10. 风险 / 待确认
   - 核心文案是否已定稿
   - 是否有真实案例、Logo、数据可用

### 宣传类页面可选 Scrollytelling 交付方案

- 适用场景
  - 宣传类页面、品牌故事页、周年页、发布页、纪录片式专题页
  - 目标不是“尽快扫完信息”，而是让用户沿着一条叙事线逐步深入理解品牌、理念、产品或历程
- 叙事方式
  - 页面采用 `Scrollytelling`（滚动叙事）手法，像纪录片一样随着滚动逐章展开
  - 推荐叙事顺序：创立初心 → 设计哲学 / 方法论 → 关键作品 / 里程碑 → 当下主张 / CTA 收口
  - 每一屏只承担一个核心信息点，避免一屏同时塞满标题、说明、指标和按钮
- 技术实现
  - 默认使用 `GSAP + ScrollTrigger`
  - `GSAP` 负责时间轴、分层 reveal、pin、parallax、媒资切换
  - `ScrollTrigger` 负责滚动位置与视觉节奏绑定
  - 若只是轻量滚动进入动效，不必强上整套 Scrollytelling
- 视觉与交互要求
  - 文字、图片、图形、年份、引言等元素应沿“故事发现线”层层展开
  - 可以使用章节式分屏、pin 场景、时间轴、画册式大图切换，但必须保证主焦点单一
  - 每一章结束都要回答一个问题：用户此刻比上一屏多理解了什么
- 响应式与降级
  - H5 不强求桌面同等滚动编排，可降级为顺序堆叠 + 轻量 reveal
  - `prefers-reduced-motion` 下应取消重 pin、长时间轴和强视差，保留静态信息可读性
  - Pad 端优先保留叙事顺序，不强保桌面镜头语言
- 使用边界
  - 这是宣传类页面的可选交付方案，不是默认模板
  - 当页面首要目标是转化效率、信息检索或后台操作时，不应滥用 Scrollytelling
  - 若内容素材不足、章节逻辑不成立，宁可回退常规高质量宣传页，也不要硬做滚动叙事

- 案例:

```text
页面采用 Scrollytelling（滚动叙事）手法，基于 GSAP + ScrollTrigger 实现。页面像一部纪录片，随着用户滚动，文字、图片、视觉元素沿着一条“故事发现线”层层展开，带领用户从创立初心进入设计哲学，再到关键里程碑，最后自然收束到 CTA。H5 端允许降级为顺序堆叠 + 轻量 reveal，避免强行保留桌面级重滚动编排。
```

## 宣传类页面规则

- 首屏必须有明确视觉焦点。
- 文案层级必须在 3 秒内能被读懂。
- 避免“居中大标题 + 三卡片”默认模板。
- 记忆点优先放在 Hero、转场、关键内容切换处。
- 动效是强化叙事，不是掩盖内容空洞。
