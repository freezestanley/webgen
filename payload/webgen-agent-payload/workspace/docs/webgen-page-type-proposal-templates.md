# webgen 页面类型方案模板汇总

> 适用场景：输出 `Discovery / Proposal` 时需要快速复用页面类型方案骨架
> 推荐顺序：先看 `docs/webgen-design-cheatsheet.md` 判断页面角色，再到本页取模板，再按需回看 `docs/design/*.md`

## 本页只回答什么

- Landing / 工具后台 / 登录表单分别该怎么起方案
- 哪些段落可以作为可直接复用方案片段
- 哪些页面可以按需升级为 `Scrollytelling`

## 1. Landing Page 方案模板

1. 页面类型 / 目标
   - 单页转化型 Landing，目标是让用户在首屏 `3` 秒内理解价值，并完成注册 / 留资 / 咨询。
2. 目标用户 / 使用场景
   - 首次访问用户，以移动端和桌面浏览为主，决策时间短，需要快速建立信任。
3. 主要板块与信息层级
   - Hero
   - 核心卖点
   - 场景 / 功能说明
   - 社会证明 / 案例
   - FAQ
   - CTA 收口
4. 设计拨盘
   - `DESIGN_VARIANCE = 6-8`
   - `MOTION_INTENSITY = 4-6`
   - `VISUAL_DENSITY = 3-4`
5. 配色 / 字体 / 图标
   - 中性底色 + `1` 个主强调色
   - 标题使用更有识别度的无衬线
   - 图标默认 `Lucide`
6. 素材 / 文案 / 接口
   - 用户提供文案优先；无文案时先做结构占位并标明待补
   - 若有表单提交，默认考虑本地代理或 mock
7. 技术实现
   - 优先原生或延续现有项目技术栈
   - 动效先用 CSS / Anime.js；滚动叙事再升级 `GSAP + ScrollTrigger`
8. 状态设计
   - 表单提交 `loading / success / error`
   - 无案例或无 FAQ 时给安全降级文案
9. 响应式策略
   - H5 首屏先露出标题、价值点、主 CTA
   - Pad 保持两段式信息组织，避免桌面布局硬缩
   - 触屏 hover 信息改默认可见或点击展开
10. 风险 / 待确认
   - 核心文案是否已定稿
   - 是否有真实案例、Logo、数据可用

### Landing Page 可直接复用方案片段

```text
页面类型为单页转化型 Landing。方案目标是在首屏 3 秒内让用户看懂价值点，并把主 CTA 放在首屏或首屏后一屏内完成收口。结构按 Hero、核心卖点、场景 / 功能说明、社会证明 / 案例、FAQ、CTA 收口展开。可以根据需求添加背景粒子动画，H5 首屏优先露出标题、价值点和主 CTA，Pad 保持两段式信息组织，不把桌面布局硬压缩到小屏。
```

### Landing Page 可选 Scrollytelling 交付方案

- 这是宣传类页面的**可选交付方案**，不是默认模板；宣传类页面可按需使用。
- 适用场景：品牌故事页、周年页、发布页、纪录片式专题页。
- 叙事顺序建议：`创立初心 → 设计哲学 / 方法论 → 关键作品 / 里程碑 → 当下主张 / CTA 收口`
- 技术实现默认使用 `GSAP + ScrollTrigger`
- H5 端允许降级为顺序堆叠 + 轻量 reveal，避免强行保留桌面级重滚动编排

### Scrollytelling proposal 片段

```text
页面采用 Scrollytelling（滚动叙事）手法，基于 GSAP + ScrollTrigger 实现。整体叙事按“创立初心 → 设计哲学 / 方法论 → 关键作品 / 里程碑 → 当下主张 / CTA 收口”推进，像纪录片一样随滚动逐章展开。H5 端允许降级为顺序堆叠 + 轻量 reveal，保证信息可读性优先。
```

## 2. Dashboard / 工具后台方案模板

1. 页面类型 / 目标
   - 工具型后台页面，目标是让用户快速读状态、完成操作、减少误判。
2. 目标用户 / 使用场景
   - 高频使用的熟练用户，桌面优先，但 Pad 也需可操作。
3. 页面布局的壳
   - 可折叠左侧多级导航
   - 固定顶部栏
   - 动态面包屑
   - 内部滚动主内容区
   - 移动端抽屉侧栏
   - 通知面板
   - 用户菜单
   - 示例路由占位页面
4. 设计拨盘
   - `DESIGN_VARIANCE = 4-6`
   - `MOTION_INTENSITY = 2-4`
   - `VISUAL_DENSITY = 6-8`
5. 配色 / 字体 / 图标
   - 中性灰阶统一，强调色只承担状态提示或主操作
   - 主界面不用 Serif
   - 数字密集区可搭配等宽数字
6. 素材 / 文案 / 接口
   - 优先读取真实字段与状态枚举
   - 接口未就绪时先定义 mock 数据结构
7. 技术实现
   - 延续现有组件系统和状态管理
   - 满屏布局优先固定页面壳体，并为主列表区、消息区、详情区定义独立滚动容器
   - 抽屉、弹窗、侧滑面板优先承担详情查看、补充编辑、辅助操作
8. 状态设计
   - 列表 `loading / empty / error`
   - 操作按钮 `pending / success / failure`
   - 过滤条件、分页、无权限、无结果要有反馈
9. 响应式策略
   - Pad 竖屏优先改为分段堆叠，而不是硬保多栏
   - 触屏主操作区按钮热区不小于 `44 x 44px`
   - 空间不足时优先折叠侧栏、筛选栏、详情栏，不强行压缩主内容区
10. 风险 / 待确认
   - 哪些字段是主决策信息
   - 哪些操作高风险，需要二次确认或撤销

### Dashboard / 工具后台可直接复用方案片段

```text
页面类型为工具型后台。方案以固定页面壳体为前提，必须先明确可折叠左侧导航、固定顶部栏、动态面包屑、内部滚动主内容区和移动端抽屉侧栏。主目标不是炫技，而是快速读状态、完成操作、减少误判；高频内容区默认拥有独立滚动容器，详情和补充操作优先用抽屉、弹窗、侧滑面板承接。
```

## 3. 登录 / 表单页方案模板

1. 页面类型 / 目标
   - 登录 / 注册 / 提交表单页，目标是低阻力完成输入与提交。
2. 目标用户 / 使用场景
   - 新用户或回访用户，常见于移动端单手操作与桌面快速录入。
3. 主要板块与信息层级
   - 标题与说明
   - 表单主体
   - 辅助说明 / 帮助入口
   - 次级操作：忘记密码、切换登录方式、返回
4. 设计拨盘
   - `DESIGN_VARIANCE = 3-5`
   - `MOTION_INTENSITY = 1-3`
   - `VISUAL_DENSITY = 4-5`
5. 配色 / 字体 / 图标
   - 保证对比度和可读性优先
   - 图标只辅助输入语义，不承担核心说明
6. 素材 / 文案 / 接口
   - 错误文案、校验规则、接口返回码需要尽量明确
7. 技术实现
   - 先保证原生可提交与校验反馈
   - 视觉动效只做轻反馈，不做重表演
8. 状态设计
   - 字段级错误
   - 提交中
   - 提交失败
   - 提交成功 / 下一步引导

### 登录 / 表单页可直接复用方案片段

```text
页面类型为登录 / 注册 / 提交表单页。方案重点是低阻力完成输入与提交，结构按标题与说明、表单主体、辅助说明 / 帮助入口、次级操作组织。默认保证字段级错误、提交中、提交失败、提交成功 / 下一步引导四类状态齐全；视觉动效只做轻反馈，不做重表演。
```

## 使用说明

- 本页是汇总模板，细则以 `docs/design/Landing.md`、`docs/design/admin.md`、`docs/design/form.md` 为准。
- 若项目已有固定品牌、组件体系或后端约束，本页只提供结构骨架，不覆盖项目自身规范。
