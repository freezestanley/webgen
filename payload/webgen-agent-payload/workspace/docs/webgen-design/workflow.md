# webgen 页面设计标准流程

> 适用场景：需要按阶段推进页面方案、实现、验证与交付
> 何时阅读：准备开工、复杂任务拆解、交付前对阶段时
> 上游文档：[`core-rules.md`](./core-rules.md)、[`index.md`](./index.md)
> 下游文档：[`skills-map.md`](./skills-map.md)、[`checklist.md`](./checklist.md)

## 本页只回答什么

- 页面任务从识别到交付，按什么顺序推进
- 每个阶段的输入、输出和完成判断是什么

## 标准流程

### 初始化: 读取页面类型方案 
- 读取页面类型初始化方案 `./page-type-entry.md`

### 阶段 A：任务识别

- 目标：判断任务属于新页面、旧页面改版、局部区块优化、视觉升级、动效增强、品牌表达加强或工具界面优化
- 输入：用户需求、现有项目上下文
- 输出：任务类型
- 完成判断：知道本轮是从 0 到 1、局部修改还是高审美升级

### 阶段 B：上下文收集

- 目标：补齐页面目标、对象、气质、技术栈、是否需要本地预览、是否允许特定动效库、是否要交付打包产物、文案/图片/API/品牌规范是否齐全
- 输入：用户消息、现有项目文档
- 输出：足够进入方案阶段的信息
- 完成判断：能回答“做给谁、解决什么、用什么做、缺什么素材”

### 阶段 C：设计定向

- 目标：确定本次偏向转化、品牌、产品效率、叙事展示、视觉冲击或沉浸式动效
- 输入：阶段 B 结果
- 输出：`Design Read`、三拨盘、`Atmosphere Layer`
- 完成判断：能说明结构变化度、视觉密度、动效强度和页面主角色

高审美页面额外要求：

- 模板默认页只保留 `hello world` 级空壳
- 先由 `design-taste-frontend` 生成初始蓝图
- 将 `Design Read`、三档位和初始蓝图摘要写入 `DISCOVERY.md`
- 补一组老板审版五问：首屏焦点、3 秒理解、证明区可信度、内容节奏、CTA 收口

### 阶段 D：初始蓝图生成

- 目标：生成区块级结构骨架、首屏与关键区块层级、布局节奏、气氛层策略、微交互方向、跨设备差异
- 输入：阶段 C 的设计定向
- 输出：可评审的初始蓝图
- 完成判断：不依赖模板默认页，也能清楚说明页面结构和视觉方向

### 阶段 E：页面质量体检

- 目标：系统检查可访问性、响应式、性能、视觉层级、信息节奏、交互反馈和边缘情况
- 输入：初始蓝图或首版页面
- 输出：问题列表
- 完成判断：知道下一轮要进入哪个专项优化，而不是笼统“再润一下”

### 阶段 F：精准精雕细琢

- 目标：按问题类型进入专项优化
- 输入：阶段 E 的问题列表
- 输出：布局、排版、色彩、细节、动效、稳健性层面的修正方案
- 完成判断：问题已经明确路由到 `arrange / typeset / colorize / polish / animate / harden`

### 阶段 G：实现策略确定

- 目标：确认是否延续现有结构、是否新增依赖、使用原生实现还是组件动效、时间线动效、3D 特效
- 输入：蓝图、质量体检结论、项目栈
- 输出：实现路径
- 完成判断：依赖和实现方式都已明确，不靠临场拼接

### 阶段 H：skill 选择

- 目标：把任务路由到正确 skill
- 输入：阶段 G 的实现路径
- 输出：skill 调用顺序
- 完成判断：知道先做蓝图、再做体检、再做专项优化、最后落实现

### 阶段 I：实现与迭代

- 目标：按骨架、层级、状态、动效、响应式、性能的顺序推进
- 输入：已确认方案和实现策略
- 输出：真实可运行页面
- 完成判断：页面主体、关键交互、核心状态和基础适配都已具备

### 阶段 J：验证与设计验收

- 目标：确认入口、资源、预览、响应式、动效、图标、技术选型与修改说明都正确
- 输入：实现结果
- 输出：验证记录
- 完成判断：
  - 至少完成一次页面实看设计复核
  - 至少检查 PC 与 H5，必要时补 Pad
  - 用户明确要求截图时，再补 CDP 截图验收

### 阶段 K：交付

- 目标：整理修改说明、风险、假设、交付路径和发布状态
- 输入：已通过验证与设计复核的页面
- 输出：可交付结果
- 完成判断：用户可直接继续验收、预览或发布决策

## 例外情况

- 复杂或大需求默认先走 `superpowers` plan，再拆子任务推进
- 若用户明确要求截图验收，再执行 CDP 访问与截图落盘
