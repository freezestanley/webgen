# webgen 禁止项与降级规则

> 适用场景：需要快速排除低质量实现与不合适的视觉方案
> 何时阅读：设计定向、实现中、设计复核时
> 上游文档：[`design-principles.md`](./design-principles.md)
> 下游文档：[`checklist.md`](./checklist.md)

## 本页只回答什么

- 默认禁止哪些常见坏味道
- 出现性能或可用性压力时如何降级

## 默认禁止项

- 不默认混用多套 icon 库
- 不默认同时混用多套动画引擎
- 不为了“酷炫”引入明显拖慢页面的背景特效
- 不使用泛滥玻璃、泛滥阴影、泛滥渐变
- 不使用无意义三列等宽卡片功能区
- 不使用与页面目标无关的 3D 特效
- 不在移动端默认保留桌面级重动效
- 不默认使用 `Inter` 作为所有项目的审美安全答案
- 不默认使用紫蓝霓虹 AI 风

## 降级规则

- 表单、转化、功能操作页面：视觉特效优先降级，交互清晰度优先
- 移动端性能不足：优先关闭粒子、模糊、复杂时间线、大面积视差
- 页面信息层级已足够强时：不再额外堆叠动画
- 外部库引入成本高于效果收益时：优先回退到原生 CSS / JS 动画
- 连续动画影响电量或滚动表现时：优先减少数量与频率

## 例外情况

- 项目已有固定品牌语言和视觉组件时，延续项目体系优先
- 用户明确要求实验性页面时，可以提高视觉戏剧性，但仍需保证可读性和降级方案
