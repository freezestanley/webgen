# webgen 页面设计总原则

> 适用场景：需要确定风格方向、版式强度、排版和视觉语言
> 何时阅读：出设计方案、做初始蓝图、修正模板味时
> 上游文档：[`core-rules.md`](./core-rules.md)、[`index.md`](./index.md)
> 下游文档：[`motion-selection.md`](./motion-selection.md)、[`anti-patterns.md`](./anti-patterns.md)

## 本页只回答什么

- 页面应如何建立信息与视觉秩序
- 如何避免模板味，稳定做出有节制的设计结果

## 核心原则

### 页面设计不是“好看”，而是信息与行为设计

页面设计必须同时负责：

- 信息层级
- 可读性
- 交互反馈
- 响应式布局
- 视觉秩序
- 性能体验
- 目标转化或任务完成率

### 高级感不靠堆元素

页面需要“高级感”时，优先依靠：

- 留白
- 比例
- 排版
- 节奏
- 色彩克制
- 材质细节
- 有节制的动效

不是依靠：

- 泛滥渐变
- 泛滥阴影
- 泛滥毛玻璃
- 大量装饰图标
- 无意义数据卡片
- 夸张炫光

### 先确定页面角色，再决定视觉强度

常见页面角色：

- 转化型：强调清晰、可信、低阻力
- 品牌型：强调记忆点、气质、统一世界观
- 产品型：强调结构、反馈、信息效率
- 叙事型：强调节奏、滚动、镜头感
- 工具型：强调秩序、密度、状态可读性

不同角色决定：

- 是否允许强视觉
- 是否需要复杂动效
- 是否强调沉浸感
- 信息密度应高还是低

## 三个设计拨盘

默认内部基线：

- `DESIGN_VARIANCE = 8`
- `MOTION_INTENSITY = 6`
- `VISUAL_DENSITY = 4`

使用方式：

- 用户明确要求“极简 / 商务 / 稳重”时，下调变体和动效强度
- 用户明确要求“实验感 / 高级感 / 记忆点 / 杂志感”时，提高结构变化和材质表达
- 工具后台默认提高信息秩序，降低视觉噪声

## 反 AI 模板味规则

默认主动规避以下俗套：

- 首屏永远居中大标题 + 按钮 + 三张等宽卡片
- 紫蓝霓虹渐变 + 发光按钮
- 大面积空泛口号文案
- 三列完全对称功能卡片
- 过多圆角、阴影、玻璃拟态堆叠
- 靠“很大标题”而不是层级建立视觉重点

替代策略：

- 首页优先考虑左文右图、分屏、错位、非完全对称布局
- 功能区避免机械 3 等分，优先 2+1、不等比、交错节奏
- 记忆点优先来自一个强场景，而不是每个区块都在抢戏

## 版式、排版与视觉规则

### 版式

- 当设计变化度较高时，默认避免首屏完全居中结构
- `md` 及以上可使用不对称布局，`md` 以下必须回落为单列安全布局
- 容器宽度默认控制在 `max-w-[1400px]` 或 `max-w-7xl`
- 复杂多列布局优先 `Grid`
- Hero、首屏、全高区块禁止默认使用 `h-screen`，改用 `min-h-[100dvh]`

### 排版

- 标题优先使用更有识别度的无衬线字体，不默认使用 `Inter`
- 偏高级 / 创意 / 品牌感场景，优先考虑：
  - `Geist`
  - `Outfit`
  - `Cabinet Grotesk`
  - `Satoshi`
- 工具型 / SaaS / Dashboard 场景禁止使用 Serif 作为主界面字体
- 正文宽度默认不超过 `65ch`
- 标题层级靠字号、字重、间距、明度建立，不靠盲目放大

### 色彩

- 单页面默认 `1` 个主强调色
- 强调色饱和度受控，不追求刺眼高纯度
- 默认禁用“AI 紫蓝发光风”
- 灰阶基底必须统一，不在同一页面里混用冷暖灰逻辑
- 若无品牌色，优先使用中性底色 + 单一高对比强调色

### 卡片、材质与阴影

- 卡片只在“层级需要被抬起”时使用
- 高密度后台默认减少卡片，优先使用分组留白、`border-t`、`divide-y`、弱分隔线
- 若使用毛玻璃，不只写 `backdrop-blur`，还应补内描边、轻微内阴影、受控透明度
- 阴影要轻、宽、染背景色，不用廉价黑色重阴影

### 状态设计

任何可交互页面，默认补齐：

- Loading
- Empty
- Error
- Success / Active Feedback

要求：

- Loading 优先骨架屏，不优先转圈 spinner
- Error 就地提示，不藏在角落
- 表单错误文案放在输入项下方
- 按钮、卡片、Tab 需要有明确按压反馈

### 动效工程

- 连续动画、磁吸、跟随、布局转场，不用粗暴 `useState` 驱动高频刷新
- 复杂持续动画要隔离到最小客户端组件中
- 动画优先只动 `transform`、`opacity`
- 禁止默认动 `top`、`left`、`width`、`height`
- 滚动复杂联动优先 GSAP / ScrollTrigger，不手写原始 scroll 监听硬堆逻辑

## 例外情况

- 既有品牌或设计系统已固定时，优先延续现有字体、色彩和版式体系
- 工具型后台可以降低结构变化度，但不能放弃状态可读性和秩序感
