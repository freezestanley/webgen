# webgen 响应式与交付规则

> 适用场景：需要确认跨设备体验、图片校验和交付前素材要求
> 何时阅读：方案阶段、实现阶段、交付前
> 上游文档：[`core-rules.md`](./core-rules.md)、[`workflow.md`](./workflow.md)
> 下游文档：[`checklist.md`](./checklist.md)

## 本页只回答什么

- 页面方案必须如何覆盖 PC / Pad / H5
- 图片和交付素材在验证阶段如何检查

## 核心规则

页面方案必须明确：

- 主要断点策略
- 触控热区策略
- Pad 横竖屏处理
- H5 首屏信息优先级
- Hover 在触屏设备上的替代方式

默认要求：

- PC / Pad / H5 都可用
- `md` 以下优先安全单列布局
- 触屏设备不依赖 hover 才能完成核心操作
- 首屏核心信息在 H5 中优先露出，不被装饰内容挤掉
- 触控热区足够大，按钮与卡片点击区域明确

## 推荐量化标准

- H5：`< 768px`
- Pad：`768px - 1279px`
- PC：`>= 1280px`
- 触控热区默认不小于 `44 x 44px`
- 正文默认控制在 `45ch - 65ch`，长文不超过 `75ch`

H5 首屏默认应在不滚动时露出：

- 页面标题或核心价值点
- `1` 个主 CTA 或主要操作入口
- 关键首图或关键状态

触屏设备上的 hover 替代方式必须落到真实交互：

- 改为点击展开
- 改为默认可见
- 改为首层直接展示关键操作

Pad 横竖屏都要验证：

- 横屏不出现内容稀疏到失衡
- 竖屏不出现桌面布局硬缩放、横向滚动或点击拥挤

## 图片校验与交付要求

- 外链真实图至少校验返回 `200` 且 `content-type` 为 `image/*`
- 预览中确认图片实际加载成功，不允许破图交付
- 若暂时无法可靠自动核对贴题性，在 `ASSETS.md` 标记“待人工核对”，并在预览时提醒用户确认

## 交付侧注意点

- 若交付物依赖外网热链，必须在交付说明中写清
- 若要求完全自托管，需要把图片下载进项目资产目录并改引用
- 若用户明确要求截图验收，再执行一次 CDP 截图

## 例外情况

- 若项目已有固定断点系统，优先沿用现有断点
- 若需求明显偏阅读型长文，可适当放宽正文宽度，但要保持阅读节奏
