# webgen 页面类型方案入口

> 适用场景：需要快速找到页面方案模板
> 何时阅读：开始出方案前
> 上游文档：[`index.md`](./index.md)
> 下游文档：`docs/webgen-page-type-proposal-templates.md`、`docs/design/*.md`

## 本页只回答什么

- 不同页面类型应先看哪个模板
- 模板的推荐使用顺序是什么

## 模板入口

- 宣传类页面、Landing 页面方案：
  - 先看 [`docs/design/Landing.md`](../design/Landing.md)
- 后管系统页面方案：
  - 先看 [`docs/design/admin.md`](../design/admin.md)
- 表单填写、登录、注册、提交流：
  - 先看 [`docs/design/form.md`](../design/form.md)

完整模板汇总仍在：

- [`docs/webgen-page-type-proposal-templates.md`](../webgen-page-type-proposal-templates.md)

## 推荐使用顺序

1. 先用 `docs/webgen-design-cheatsheet.md` 判断页面角色和风格方向
2. 再看对应页面类型模板
3. 最后回到 [`core-rules.md`](./core-rules.md) 和 [`workflow.md`](./workflow.md) 对齐硬规则与阶段顺序

## 例外情况

- 项目已有固定品牌或组件体系时，模板只作为结构参考，不覆盖项目自身规范
