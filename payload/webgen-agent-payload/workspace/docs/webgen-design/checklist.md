# webgen 页面交付前检查清单

> 适用场景：页面交付、验收、设计复核前的最终检查
> 何时阅读：实现完成后、宣称验证完成前
> 上游文档：[`workflow.md`](./workflow.md)、[`responsive-delivery.md`](./responsive-delivery.md)
> 下游文档：无

## 本页只回答什么

- 交付前最低限度要检查什么
- 哪些检查属于运行验证，哪些属于设计复核

## 基础运行检查

- 页面入口是否正常
- 静态资源是否正确引用
- 本地预览是否可运行
- 主要断点下布局是否正常
- PC / Pad / H5 是否都能使用
- 若存在大文件风险，是否已先拆成多个小份文件并分步写入，而非整页灌入单文件

## 交互与状态检查

- 文字可读性是否足够
- 图标是否统一为 `Lucide` 或项目既有图标体系
- `Loading / Empty / Error / Active` 状态是否齐全
- 主要交互元素是否具备键盘可达、可见焦点、可点击反馈
- 触屏主操作是否不依赖 hover
- 若页面为表单 / 后台 / 工具型：错误提示是否在控件附近可见

## 动效与性能检查

- 动效是否与场景匹配
- 动画库是否选型正确
- 页面是否因特效牺牲性能
- 是否有必要的降级处理
- 若使用动画：
  - 主要动画不依赖 `top / left / width / height`
  - 有移动端降级或 `prefers-reduced-motion` 处理
- 若使用外部动效库：
  - 已说明为什么不用原生 CSS / JS 即可解决
  - 已说明为什么该库比同页其他方案更合适

## 设计复核检查

- 是否存在明显 AI 模板味
- 若页面为阅读型或营销型：
  - 首屏 `3` 秒内能读懂核心价值点
  - 主 CTA 在首屏或首屏后一屏内明确可见
- 高审美页面必须至少完成一次页面实看设计复核
- 用户明确要求截图时，才追加 CDP 截图验收

## 交付说明检查

- 是否说明了实现取舍与假设
- `DISCOVERY.md / Proposal` 是否已明确记录实现拆分计划与 `80%` compact 策略
- 实现过程中若上下文占用估算达到 `80%`，是否已先 compact 再继续
- 图片是否完成校验或在 `ASSETS.md` 标记待人工核对
- 若依赖外网热链，是否在交付说明中注明

## 例外情况

- 若体检或设计复核未通过，必须继续优化后复验
- 若风格已经稳定，建议在 `skills/impeccable` 体系内执行 `teach-impeccable`
