# webgen 动效与特效技术选型

> 适用场景：需要为页面选择动画库或特效实现方式
> 何时阅读：准备加动效、滚动叙事、复杂转场、3D 效果前
> 上游文档：[`design-principles.md`](./design-principles.md)、[`skills-map.md`](./skills-map.md)
> 下游文档：[`responsive-delivery.md`](./responsive-delivery.md)

## 本页只回答什么

- 当前场景该用哪套动效技术
- 哪些情况下应降级或避免混用

## 选择原则

所有动效必须服从：

- 可读性
- 性能
- 响应式适配
- 交互清晰度
- 设备兼容与降级策略

## 场景到技术的决策表

### `Anime.js`

适合：

- 普通 UI 入场
- 小交互
- 小范围强调

不适合：

- 长滚动叙事
- 多段复杂时间线

### `Motion`

适合：

- React 组件交互与状态动画
- 组件级转场
- 局部反馈动画

不适合：

- 大型滚动驱动编排
- 页面级长时间线

### `GSAP`

适合：

- 滚动驱动
- 时间线编排
- 多元素联动
- 叙事动画

使用规则：

1. `gsap-core` 处理基础动画属性和 easing
2. `gsap-timeline` 处理多步骤编排
3. `gsap-scrolltrigger` 处理滚动驱动
4. React 项目默认补看 `gsap-react`
5. 大滚动场景上线前默认补看 `gsap-performance`

### `Three.js`

适合：

- 3D 场景
- 粒子效果
- 空间视觉
- 沉浸式特效

前提：

- 必须附带移动端降级策略
- 不得牺牲主要信息可读性

## 动效硬规则

1. 普通 UI 入场、小交互、小范围强调：优先 `Anime.js`
2. React 组件交互与状态动画：优先 `Motion`
3. 滚动驱动、时间线编排、叙事动画：优先 `GSAP`
4. 3D 场景、粒子、空间视觉、沉浸式特效：优先 `Three.js`
5. 需要更有设计感的组件效果、但当前模板不是 React 时：优先把 `React Bits` 当作参考源，再用当前技术栈复刻
6. 单页不无原则混用过多动画库
7. 同一组件树内，避免把 GSAP / Three 与组件级 Motion 胡乱混在一起
8. 允许更强视觉戏剧性，但重点只打造 `1` 个强记忆场景

## 降级与收口

- 表单、转化、功能操作页面：视觉特效优先降级，交互清晰度优先
- 移动端性能不足：优先关闭粒子、模糊、复杂时间线、大面积视差
- 页面信息层级已足够强时：不再额外堆叠动画
- 外部库引入成本高于效果收益时：优先回退到原生 CSS / JS 动画
- 连续动画影响电量或滚动表现时：优先减少数量与频率

## 例外情况

- 项目现有体系已固定某套动效方案时，优先延续既有技术
- 若用户明确指定技术路线，可按指定方案做，但仍需给出降级策略
